## OAuth 프로바이더

### Google OAuth
```javascript
signInWithOAuth({
  provider: 'google',
  options: {
    redirectTo: `${siteUrl}/`,
    queryParams: { access_type: 'offline', prompt: 'consent' }
  }
})
```

### 카카오 OAuth
```javascript
signInWithOAuth({
  provider: 'kakao',
  options: {
    redirectTo: `${siteUrl}/`,
    scopes: 'profile_nickname profile_image account_email'
  }
})
```

## 인증 플로우

1. 사용자가 로그인 페이지에서 Google/카카오 버튼 클릭
2. Supabase가 해당 OAuth 프로바이더로 리다이렉트
3. 사용자가 프로바이더에서 인증 완료
4. Supabase 콜백으로 리다이렉트 (PKCE 코드 교환)
5. AuthContext가 세션 변경 감지하여 user 상태 업데이트
6. Navbar에 사용자 아바타/이름 표시

## AuthContext (src/contexts/AuthContext.jsx)

### 제공 값
| 값 | 설명 |
|----|------|
| `user` | Supabase User 객체 |
| `loading` | 인증 로딩 상태 |
| `isAuthenticated` | 로그인 여부 (boolean) |
| `getUserName()` | 사용자 이름 (메타데이터에서 추출) |
| `getUserAvatar()` | 프로필 이미지 URL |
| `getUserInitial()` | 이름 첫 글자 (아바타 대체) |

### 세션 타임아웃
- 5초 내 초기 세션 확인 미완료 시 loading → false
- onAuthStateChange 리스너로 실시간 세션 추적

## Supabase 대시보드 설정 필요 사항
1. **Authentication > Providers**: Google, Kakao 프로바이더 활성화
2. **Authentication > URL Configuration**:
   - Site URL: `https://presentation.dreamitbiz.com`
   - Redirect URLs: `https://presentation.dreamitbiz.com/**`
3. **Google**: Google Cloud Console에서 OAuth Client ID/Secret 발급
4. **Kakao**: Kakao Developers에서 REST API Key 발급
