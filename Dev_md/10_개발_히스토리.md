### 개발 과정
1. **분석 단계**
   - GitHub 리포지토리(aebonlee/presentation) 분석: README + CNAME만 존재하는 빈 저장소
   - 회사 메인 사이트(D:\dreamit-web\www\react-source) 분석:
     - React 19 + Vite 7 + Supabase + React Router v7
     - 순수 CSS + CSS Variables
     - 글래스모피즘 디자인
     - Google/카카오 OAuth
     - 5가지 컬러 테마
     - 다크/라이트/오토 모드
   - 메인 사이트의 디자인 패턴, 컴포넌트 구조, 인증 방식을 참고하여 동일한 기술 스택 적용

2. **프로젝트 초기화**
   - Vite + React 프로젝트 생성
   - 디렉토리 구조 설계 (메인 사이트 패턴 참고)
   - package.json 작성 (동일 의존성)

3. **디자인 시스템 구축**
   - base.css: 다크 블루 기반 디자인 토큰 + 5가지 컬러 테마
   - 11개 CSS 파일 작성 (네비바, 히어로, 푸터, 커리큘럼, 도구, 인증 등)
   - 다크 모드 / 반응형 / 애니메이션

4. **핵심 시스템 구축**
   - Supabase 클라이언트 설정 (PKCE 인증)
   - Google/카카오 OAuth 인증
   - ThemeContext, AuthContext, ToastContext
   - AOS 커스텀 훅

5. **콘텐츠 데이터 작성**
   - 5개 커리큘럼: 총 20모듈, 60+ 토픽
   - 6개 도구: 가격표, 기능, 장단점, 팁, 비교표

6. **컴포넌트/페이지 개발**
   - Navbar: 글래스모피즘, 드롭다운, 테마/컬러 전환, 반응형
   - Footer: 4컬럼, 패밀리 사이트
   - 7개 페이지: Home, About, Login, NotFound, CourseDetail, ToolsOverview, ToolDetail
   - SEOHead: OG/Twitter 메타태그

7. **OG 이미지 생성**
   - sharp로 SVG→PNG 변환
   - 1200x630 다크 블루 그라디언트 이미지

8. **빌드 검증**
   - npm install: 182 패키지, 취약점 0
   - npm run build: 성공 (6.92초)
   - 총 CSS 39.48KB, JS 436.31KB + 페이지별 청크

### 사용 기술 결정 이유
| 결정 | 이유 |
|------|------|
| React 19 + Vite 7 | 회사 메인 사이트와 동일 스택 |
| 순수 CSS + CSS Variables | 메인 사이트 패턴 일관성 |
| Supabase | 메인 사이트와 동일 인프라 |
| Google + 카카오 OAuth | 한국 사용자 대상 |
| sharp (OG 이미지) | 서버 없이 빌드 시 이미지 생성 |
| AOS (Intersection Observer) | 메인 사이트와 동일 애니메이션 방식 |
| Lazy Loading | 성능 최적화 (코드 스플리팅) |
