## Phase 1: 초기 개발 (`614d0bf`)

### 개발 과정
1. **분석 단계**
   - GitHub 리포지토리(aebonlee/presentation) 분석: README + CNAME만 존재하는 빈 저장소
   - 회사 메인 사이트(`D:\dreamit-web\www\react-source`) 분석: React 19 + Vite 7 + Supabase + 글래스모피즘
   - 메인 사이트의 디자인 패턴, 컴포넌트 구조, 인증 방식을 참고하여 동일한 기술 스택 적용

2. **프로젝트 초기화** — Vite + React, 디렉토리 구조 설계

3. **디자인 시스템 구축** — 다크 블루 기반 토큰, 5가지 컬러 테마, 11개 CSS 파일

4. **핵심 시스템** — Supabase PKCE 인증, ThemeContext, AuthContext, ToastContext, AOS 훅

5. **콘텐츠 데이터** — 5개 커리큘럼, 6개 도구

6. **컴포넌트/페이지** — Navbar, Footer, 7개 페이지 (Home, About, Login, NotFound, CourseDetail, ToolsOverview, ToolDetail)

7. **OG 이미지** — sharp로 SVG→PNG (1200×630)

## Phase 6: 제작도구 종합 가이드 시스템 (신규)

### 개발 배경
기존 도구 소개 페이지(`ToolDetail`)가 기능 목록, 가격, 장단점, 5개 팁 수준에 머물러 있어, 각 도구를 실제로 활용하기 위한 **종합 사용 가이드**가 필요.

### 아키텍처 결정
| 결정 | 이유 |
|------|------|
| 별도 `tool-guides/` 디렉토리 | 도구당 500~1300줄 → `tools.js`에 추가 시 5000줄+ 비대화 방지 |
| 새 `ToolGuide.jsx` 페이지 | 기존 ToolDetail은 소개 유지, 별도 가이드 페이지로 분리 |
| `renderMarkdown.js` 유틸리티 추출 | LearningPage 내부 함수 → 공유 유틸리티로 추출하여 중복 제거 |
| LearningPage 패턴 재사용 | 사이드바+섹션탭+마크다운 렌더링 구조 동일 |

### 신규 생성 파일 (10개)
```
src/utils/renderMarkdown.js          # 마크다운 렌더러 (LearningPage에서 추출)
src/styles/tool-guide.css            # 가이드 전용 CSS (~350줄)
src/pages/tools/ToolGuide.jsx        # 가이드 페이지 컴포넌트
src/data/tool-guides/index.js        # 어그리게이터 + getGuideByToolId()
src/data/tool-guides/powerpoint.js   # PowerPoint 가이드
src/data/tool-guides/google-slides.js # Google Slides 가이드
src/data/tool-guides/canva.js        # Canva 가이드
src/data/tool-guides/miricanvas.js   # 미리캔버스 가이드
src/data/tool-guides/figma.js        # Figma 가이드
src/data/tool-guides/genially.js     # Genially 가이드
```

### CSS 구조 (tool-guide.css)
- `.tool-guide-page` — 사이드바+콘텐츠 flex 레이아웃
- `.tool-guide-sidebar` — 280px sticky 사이드바, 섹션/서브섹션 네비게이션
- `.tool-guide-content` — 브레드크럼, 헤더, 섹션탭, 서브섹션탭
- `.tool-guide-markdown` — 마크다운 콘텐츠 스타일 (h3, h4, table, code, kbd)
- `.tool-guide-cta` — ToolDetail에 추가된 가이드 유도 카드 (그래디언트)
- 다크 모드 `[data-theme="dark"]` 오버라이드
- 반응형: 1024px (사이드바 토글), 768px (모바일 축소)

## 기술 결정 이유

| 결정 | 이유 |
|------|------|
| React 19 + Vite 7 | 회사 메인 사이트와 동일 스택 |

| Supabase | 메인 사이트와 동일 인프라 |
| Google + 카카오 OAuth | 한국 사용자 대상 |

| Lazy Loading | 성능 최적화 (코드 스플리팅) |
| gh-pages | 무료 정적 호스팅 + 커스텀 도메인 |
